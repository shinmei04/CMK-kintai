# 勤怠・請求メモ

GoogleスプレッドシートとGoogle Apps Scriptだけで動く、勤怠・請求集計用のWebアプリです。

レッスンは「コマ数」で記録し、講師外業務だけ「出勤」「退勤」で時間を記録します。バックエンドサーバーやnpmの環境構築は不要です。

## できること

- レッスンをコマ数で保存
- 3コマなら `CB / CS / CB` のように各コマ内容を選択
- `CB`、`CS`、`MJ`、`MS`、`CE`、`キャンプ10時` などの各コマ内容から請求分類を自動判定
- 講師外業務は `出勤` と `退勤` で時間計算
- 講師外業務のよく使う内容を最大6個まで登録
- 月別の勤務履歴を日付ごとに確認
- 履歴から講師業務と講師外業務を編集・削除
- 日をまたぐ講師外業務を開始日分の勤務として集計
- 月次集計を表示
- コピー用テキストを作成
- PCテストモードで、Apps Scriptへ貼り付ける前に画面操作を確認

## ファイル構成

```text
.
├── Code.gs
├── Index.html
└── README.md
```

- `Code.gs`: Google Apps Scriptのサーバー側処理
- `Index.html`: Webアプリの画面
- `README.md`: セットアップと使い方

## 必要なもの

- Googleアカウント
- Googleスプレッドシート

Node.js、npm、Python、外部サーバーは不要です。

## まずPCでテスト版を試す(任意)

Apps Scriptへ貼り付ける前に、画面だけ試せます。

1. GitHubのリポジトリページを開きます。
2. `Code` → `Download ZIP` を押します。
3. ZIPを展開します。
4. 展開したフォルダ内の `Index.html` をブラウザで開きます。
5. 画面上部に `PCテストモード` と表示されればOKです。

PCテストモードでは、データはブラウザ内の `localStorage` に保存されます。スプレッドシートには保存されません。

PCテストモードでも、月別履歴の表示、履歴からの編集・削除、日をまたぐ勤務時間の計算、編集後の集計反映を確認できます。

## Googleスプレッドシートを作る

1. Googleドライブを開きます。
2. `新規` → `Googleスプレッドシート` を選びます。
3. ファイル名を `勤怠・請求メモ` などに変更します。
4. メニューから `拡張機能` → `Apps Script` を開きます。

シートは初回実行時に自動作成されます。手動で作る必要はありません。

## Apps Scriptにファイルを貼り付ける

### 1. Code.gsを貼り付ける

1. Apps Scriptエディタで、左側の `Code.gs` を開きます。
2. 最初から入っているコードをすべて削除します。
3. このリポジトリの `Code.gs` の中身をすべて貼り付けます。

### 2. Index.htmlを作る

1. Apps Scriptエディタ左側の `+` を押します。
2. `HTML` を選びます。
3. ファイル名に `Index` と入力します。
4. 作成されたHTMLファイルに、このリポジトリの `Index.html` の中身をすべて貼り付けます。

Apps Script上ではファイル名が `Index` と表示されても問題ありません。

### 3. タイムゾーンを確認する

1. Apps Script左側の歯車アイコンを開きます。
2. タイムゾーンを `Asia/Tokyo` にします。
3. 保存します。

コード側でも `Asia/Tokyo` を指定しています。

## 初期設定を実行する

1. Apps Script上部の関数選択で `setup` を選びます。
2. `実行` を押します。
3. 権限確認が表示されたら、自分のGoogleアカウントを選びます。
4. `詳細` → プロジェクト名のリンク → `許可` の順に進みます。

実行後、スプレッドシートに以下のシートが作成されます。

- `勤怠ログ`
- `月次集計`

## Webアプリとしてデプロイする

1. Apps Script右上の `デプロイ` を押します。
2. `新しいデプロイ` を選びます。
3. 種類の選択で `ウェブアプリ` を選びます。
4. 説明に `初回デプロイ` などを入力します。
5. `次のユーザーとして実行` は `自分` を選びます。
6. `アクセスできるユーザー` は自分だけで使うなら `自分のみ` を選びます。
7. `デプロイ` を押します。
8. 表示されたWebアプリURLを開きます。

WebアプリURLをスマホで開くと、そのままスマホ用画面として使えます。

## コードを更新したとき

GitHubから新しい `Code.gs` や `Index.html` を取り込んだだけでは、公開済みWebアプリには反映されません。

1. Apps Scriptで `Code.gs` と `Index.html` を貼り替えます。
2. 保存します。
3. `デプロイ` → `デプロイを管理` を開きます。
4. 対象デプロイの鉛筆アイコンを押します。
5. バージョンで `新バージョン` を選びます。
6. `デプロイ` を押します。

## スプレッドシートの列構成

`勤怠ログ` シートは以下の列を使います。

| 列 | 項目 |
| --- | --- |
| A | ID |
| B | 種別 |
| C | 日付 |
| D | 開始 |
| E | 終了 |
| F | 内容 |
| G | 分類 |
| H | コマ数 |
| I | 分 |
| J | 時間 |
| K | メモ |
| L | 作成日時 |
| M | 更新日時 |

`月次集計` シートには、月次出力を実行した月の集計結果が書き込まれます。

## 画面構成

Webアプリは以下の3タブ構成です。

- `今日`: 今日の勤務状態、出勤・退勤、講師業務登録を行います。
- `履歴`: 対象月の勤務実績を日付ごとに確認し、行を選んで編集・削除します。
- `集計`: 対象月の通常コマ、キャンプ、講師外業務時間、要確認件数を確認し、コピー用テキストを作成します。

スマートフォンでは画面下部にタブバーが表示されます。PCでは画面上部に水平ナビゲーションが表示され、中央寄せの幅で操作できます。

## 普段の使い方

### レッスンを記録する

1. `今日` タブを開きます。
2. 日付は今日の日付で自動保存されます。
3. コマ数を選びます。
4. 表示された各コマの内容を選びます。
5. 必要なら補足メモを入力します。
6. `講師業務を登録` を押します。

新規登録と編集のコマ数は、どちらも1〜4コマです。

請求分類は各コマ内容から自動判定されます。

- `CB`、`CS`、`MJ`、`MS`、`CE`、`プラコ`、`体験会`、`振替`、`補講`、`キャンプその他`: 通常コマ
- `キャンプ10時`: キャンプ10時
- `その他・要確認`: その他・要確認

### 講師外業務を記録する

1. `今日` タブを開きます。
2. 内容、分類、メモを入力します。
3. 必要なら `よく使う内容` のボタンを押します。
4. `出勤する` を押します。
5. 終わったら `退勤` を押します。

退勤時は何も入力しません。終了時刻、分、時間が自動計算されます。

### よく使う内容を編集する

1. `出勤` を押します。
2. ポップアップ内の `よく使う内容` の右にある `編集` を押します。
3. 最大6個まで入力します。
4. `保存` を押します。

PCテストモードではブラウザ内に保存されます。Apps Script版ではユーザー設定に保存されます。

### 間違えて出勤した場合

まだ退勤していない場合は、`出勤を取り消す` を押します。未退勤の出勤ログが削除されます。

退勤済みのログや講師業務を直す場合は、`履歴` タブで該当行を選び、編集シートから保存してください。

### 月別履歴を確認する

1. `履歴` タブを開きます。
2. 右上の対象月を選びます。
3. 勤務実績が日付ごとに表示されます。
4. 講師外業務は内容、開始時刻、終了時刻、合計時間を確認できます。
5. 講師業務は各コマの内容とコマ数を確認できます。

### 勤務実績を編集する

1. `履歴` タブで編集したい勤務実績の行を選びます。
2. スマートフォンでは下から編集シートが開きます。PCでは中央のモーダルとして開きます。
3. 講師外業務は日付、開始時刻、終了時刻、内容、分類、メモを編集できます。
4. 講師業務は日付、コマ数、各コマの内容、メモを編集できます。
5. `完了` を押すと保存され、履歴・今日の勤務状態・集計が更新されます。

講師外業務の開始時刻または終了時刻を変更すると、編集画面内で合計勤務時間が再計算されます。

### 勤務実績を削除する

1. `履歴` タブで削除したい勤務実績の行を選びます。
2. 編集シート下部の `この勤務実績を削除` を押します。
3. 確認ダイアログで確定すると、履歴・今日の勤務状態・集計が更新されます。

削除対象はログIDで特定されます。

## 月末の使い方

1. `集計` タブを開きます。
2. 右上の対象月を選びます。
3. 通常コマ、キャンプ、講師外業務時間、要確認件数を確認します。
4. 必要なら `コピー用テキストを作成` を押します。
5. 作成されたテキストを申請フォームなどに貼り付けます。

## 集計ルール

### レッスン

- 種別が `レッスン` の行を対象にします。
- `通常コマ` は通常コマ数に合算します。
- `キャンプ10時` はキャンプ10時コマ数に合算します。
- `その他・要確認` は通常コマには含めず、要確認として表示します。
- `レッスン合計コマ数` は `通常コマ数 + キャンプ10時コマ数` です。

### 講師外業務

- 種別が `講師外業務` の行を対象にします。
- 分を合計し、60で割って時間を出します。
- 時間は小数第2位で表示します。
- 終了時刻が空欄の行は、終了漏れとして警告します。
- 終了時刻が開始時刻より前の場合は、翌日の終了時刻として計算します。
- 月次集計では、開始日を勤務日として扱います。

## よくあるトラブル

### Webアプリに変更が反映されない

Apps Scriptはデプロイ済みバージョンを表示します。コードを貼り替えたあと、必ず新しいバージョンで再デプロイしてください。

### 権限エラーが出る

Webアプリの実行ユーザーが、対象スプレッドシートを編集できるか確認してください。基本設定は `次のユーザーとして実行: 自分` がおすすめです。

### 開始しっぱなしになった

そのまま終わらせる場合は `退勤` を押します。間違えて出勤しただけなら `出勤を取り消す` を押します。

### 分類や内容を間違えた

`履歴` タブで該当行を選び、編集シートから修正してください。必要な場合だけスプレッドシートの `勤怠ログ` シートを直接確認してください。

### PCテストモードのデータを消したい

ブラウザの開発者ツール、またはサイトデータ削除で `localStorage` を削除してください。わからない場合は、別のブラウザで `Index.html` を開くと空の状態で試せます。

## Google Apps Scriptへの反映方法

このリポジトリの変更をApps Script本番環境へ反映するには、`Code.gs` と `Index.html` をApps Scriptエディタへ貼り替え、保存後に新しいバージョンで再デプロイしてください。GitHub上のファイルを更新しただけでは、公開済みWebアプリには反映されません。

## 動作確認チェックリスト

1. PCで `Index.html` を開き、PCテストモードになることを確認します。
2. `今日`、`履歴`、`集計` タブを切り替えます。
3. レッスンでコマ数を `3` にします。
4. 各コマで `CB`、`CS`、`CB` を選びます。
5. `講師業務を登録` を押します。
6. 内容を入力して `出勤する` を押します。
7. 勤務中の経過時間が更新されることを確認します。
8. `退勤する` を押します。
9. `履歴` タブで対象月の勤務実績が日付ごとに表示されることを確認します。
10. 講師外業務を開き、終了時刻を開始時刻より前にして日をまたぐ時間が計算されることを確認します。
11. 講師業務を開き、コマ数と各コマ内容を編集できることを確認します。
12. 勤務実績を削除できることを確認します。
13. `集計` タブで編集・削除結果が反映されることを確認します。
14. `コピー用テキストを作成` が動くことを確認します。

Apps Script版では、同じ操作をしたあと `勤怠ログ` と `月次集計` に反映されることも確認してください。
